# 深层动力框架（Depth Dynamics）

> 回答其他框架不回答的问题：**他为什么会长成这样？**
> motives 框架告诉你他的主导动机是 nAch；本框架追问：nAch 在他身上为什么是这个形态？
> 它防御着什么？它的能量从哪里来？代价由人格的哪个部分支付？

**定位**：本框架处于推断链的最深层，输入是其他框架的输出 + 传记事实，
不直接从单条语料编码。**必须在 motives / values-hierarchy / cit 完成后执行。**

**伦理与置信度约束**：本框架结论天然处于"中置信"上限，发生学假设一律标"低置信/推测"。
禁止临床诊断语言（"自恋型人格""回避型依恋"等 DSM/依恋术语），用行为描述语言替代。

---

## 维度 0：生成机制（Generative Engine）——本框架的核心交付物

**问题**：能否找到**一台引擎**，把此人的全部稳定行为模式作为输出推导出来？

维度 1–5 产出的是清单（支柱、恐惧、冲突各一条），清单**描述**人；
只有当它们被组装成一条因果链时，分析才开始**解释**人。本维度强制完成这次组装。

**引擎的标准形态（五环链）**：

```
原初情境 → 核心张力 → 补偿策略 → 强化循环 → 行为输出族
```

1. **原初情境**：成长期**反复出现的情绪处境**（不是单一事件，是一种长期身位）。
   例：智力优越 × 资源劣势的双重身位；被寄予厚望 × 情感忽视的组合。
2. **核心张力**：该处境制造的不可兼得困境或慢性匮乏——人格围绕它组织起来。
3. **补偿策略**：他在原初情境中发现的**唯一可用的不对称资产**及其使用方式。
   关键洞察：成年后的"标志性策略"往往是童年生存策略的泛化。
4. **强化循环**：策略每次成功都巩固身份支柱，同时加深对替代策略的废用——
   解释为什么此人"只会这一招"且这一招越用越纯。
5. **行为输出族**：列出引擎推导出的全部观察到的模式。

**回推检验（强制，引擎的合格线）**：

- 引擎必须能推导 ≥5 个**已观察**的行为模式，其中至少 1 个是表层框架
  （big5/motives/values）**解释不了或解释得很勉强**的——引擎的价值恰恰在于
  收编表层框架的"残差"
- 必须列出引擎**解释不了**的模式（残差清单）——没有残差的引擎是过拟合的故事
- 必须做**时机检验**：引擎不仅要解释他"做了什么"，还要解释"为什么是那个时间点"。
  能解释行为却解释不了时机的引擎，权重减半

**单一引擎原则（奥卡姆）**：优先寻找一台引擎解释多数模式，而非多台小引擎各管一段。
但禁止硬塞：若证据确实需要两台引擎（如"先天认知风格"与"后天嫁接的道德框架"并存），
明确写出两台引擎的边界与交互，不强行合并。

**与素朴假设对照（强制）**：引擎是深层解释，必须与"这只是理性选择/性格使然"的
素朴假设并列进 ACH。引擎胜出的判据只有一个：**对时机和残差的解释力**。
若素朴假设同样能解释全部证据，引擎降级为"未裁决"，并给出未来可裁决的观察判据。

## 维度 1：身份支柱（Identity Pillars）

**问题**：此人的自我价值感建立在哪几根承重墙上？抽掉哪一根，人格会塌？

**操作化指标**：
- 他**从不自嘲**的领域（自我价值核心区不允许玩笑）
- 受到何种攻击时**防御反应最不成比例**（攻击触及承重墙）
- 介绍自己时**always 提及**与**从不提及**的身份要素
- 失败叙事中**被保护的部分**（"我错了，但不是因为我不够 X"——X 即支柱）

**输出**：1-3 根身份支柱，按承重排序，每根附触发证据。

**关键推论**：身份支柱 = 不可谈判区。预测此人行为时，任何要求他放弃支柱的交易，
无论补偿多大都会被拒绝；任何保护支柱的退路，无论代价多大都会被接受。

## 维度 2：核心恐惧与防御结构（Core Fear & Defenses）

**问题**：他最显眼的行为模式，是在防御什么？

**方法**：把此人的稳定行为模式逐条做"防御翻转"——假设该模式不是偏好而是防御，
问"如果不这样做，他会暴露在什么面前？"

| 常见模式 | 防御翻转假设 |
|----------|------------|
| 超然/去个人化 | 防依赖、防被评价 |
| 极致控制/亲力亲为 | 防失控、防托付后的背叛 |
| 理性化/数据化一切 | 防羞耻、防情绪暴露 |
| 提前退场/主动离开 | 防被抛弃（先走的人不会被丢下）、防衰退被目睹 |
| 永远的下一个挑战 | 防停下来之后的空洞、防"已完成"状态下的自我审视 |

**证据要求**：防御假设必须有**裂缝时刻**支撑——即防御短暂失效、被防御物露头的记录
（罕见的失态、自我矛盾的同现、不合人设的情绪流露）。无裂缝证据的防御假设标"纯推测"并降权。

**与 ACH 衔接**：每个防御假设必须与"这只是偏好/策略"的素朴假设并列进 ACH 矩阵，
防御解释**不自动优先**。

## 维度 3：核心冲突（Core Conflict)

**问题**：他身上哪两个需求不可同时满足？冲突如何被管理？

**操作化指标**：
- 同一场合先后说出的自我矛盾陈述（"想刷存在感"+"不在乎形象"）
- 言行三角验证（Step 2.5）中的**轻微分歧**条目——重大分歧多为利益驱动，
  轻微分歧才是内在冲突的指纹
- 行为上的"补偿对"：每次做 A 之后不久做反向的 B（高调之后刻意隐身等）

**输出**：核心冲突一对（最多两对），管理策略（压抑/分区/升华/外包），
以及**冲突失管的触发条件**——什么情境下管理策略会失效。

**关键推论**：人在核心冲突失管时的行为最偏离基线模型。预测台账中应有
至少一条预测针对"冲突失管情境"。

## 维度 4：自我叙事的功能分析（Narrative Function）

**问题**：他讲述自己的故事，在为他做什么工作？

不分析叙事内容的真假（那是 Step 2.5 的事），分析叙事的**功能**：
- **合法化**：为已做的决定提供道德/理性外衣
- **免责**：预先安置失败的解释框架
- **预演**：通过公开承诺把未来行为锁进轨道（自我承诺装置）
- **修复**：重写早期经历，使人生显得连贯有意

**操作化指标**：对比同一事件在**不同年份**的讲述版本，叙事修订的方向即功能所在。
（例：离开谷歌的理由在 2010 年访谈 vs 2018 年访谈中如何变化？）

## 维度 5：发生学假设（Genetic Hypothesis）——一律低置信

**问题**：什么样的成长路径会产出这套人格配置？

**前提**：仅当语料含可靠传记事实（出身、家庭、教育轨迹、早期关键人物）时执行。
无传记语料 → 整个维度标 `N/A`，禁止从成年行为反推童年。

**方法**：列出传记事实 → 提出 1-2 条形成路径假设 → 标注哪些成年模式被该假设解释、
哪些**不被解释**（不被解释的部分必须列出，防止叙事过度拟合）。

**输出格式**：「假设：[路径]。解释了 [模式A/B/C]。无法解释 [模式D]。置信度：低。」

---

## 最小引证门槛

- 身份支柱：每根 ≥2 个跨情境证据
- 防御结构：每个假设 ≥1 个裂缝时刻，否则标"纯推测"
- 核心冲突：≥1 个矛盾同现实例（同场合/同文本）
- 叙事功能：≥2 个版本的同事件讲述（无多版本 → 该维度降级为单点观察）
- 发生学：无传记语料即 N/A

```
   时机检验：引擎对 [关键决策] 时间点的解释
   残差清单：引擎解释不了的模式

```

**写作提示**：维度 1–3 的产出如果无法从维度 0 的引擎推导出来，
说明引擎找错了或维度 1–3 编码错了——回头重做，不允许并列交付互不相干的结论。

# 特稿版输出模版（Narrative Profile）

> Deep Mode 的第二输出。技术版（output-schema.md）面向程序与档案库；
> 特稿版面向人的阅读。两版结论必须一致——特稿是技术版的**翻译**，不是另一次分析。
>
> **黄金标准：《纽约客》（The New Yorker）人物特稿。** 一个完全不懂心理学、
> 从未听过本系统任何术语的普通读者，应当能从头到尾顺畅读完，并在某一刻产生
> "我好像真的看懂了这个人"的体验——而不是"我看懂了一份分析报告"。
> 衡量成稿的唯一标准是：**把它读给一个不搞这行的朋友听，他会不会走神。**

## 文件命名

`profiles/<人物>_<日期>.特稿.md`，与技术版同时产出。

## 写作总纲

1. **场景开篇**：用一个"言行错位"或"反常细节"的具体场景切入，不用框架名词开篇。
   最好的开篇细节来自言行三角验证（Step 2.5）的分歧条目。
2. **行为弧线做主线**：以 CIT 关键事件串联人物弧线（"三次离开"式结构），
   框架结论作为弧线的解释嵌入，而非独立章节。
3. **零黑话（最高优先级）**：正文里**不允许出现任何方法论或心理学术语**。
   不只是把术语"翻译得通俗一点"，而是让它**彻底消失**——读者应该体会到那个
   心理机制，却从头到尾不知道它有名字。下列词及其同类**严禁出现在正文**：
   生成机制 / 引擎 / 回推检验 / 残差 / 决定性货币 / 双重身位 / 核心张力 /
   补偿策略 / 强化循环 / 身份支柱 / 防御 / 核心冲突 / 归因 / ACH / 置信度 /
   nAch / Big Five / 框架 / 维度 / 编码 / 假设（作分析术语时）。
   - ✗（带脚手架的散文）："他的核心张力是智力优越×资源劣势的双重身位"
   - ✓（消化进故事）："论家里有钱、论见过世面，他都比不过；他手里只有一样
     东西是别人夺不走的，就是脑子。"
   - 检验：把任意一段单独拎出来，它读起来必须像《人物》或《纽约客》的句子，
     而不像一份报告里的段落。
4. **机制要被故事消化，不能被命名**：成因分析（depth 框架）是文章的灵魂，
   但它必须**化在叙事里让读者自己悟到**，而不是被讲解。高潮处读者应产生
   "原来他一辈子都在做同一个动作"的顿悟——这个顿悟由具体事例堆出来，
   不由一句"这就是他的生成机制"点破。证伪判据、证据边界、置信度分层
   **全部移到文末"关于这篇文章"附注**，正文一个字都不提。
   正文负责"让人信且让人懂"，附注负责"对得起严谨"。
5. **可读性优先于结构完整**：不必凑满固定节数；用空行分隔的自然段落群即可，
   小标题能省则省。宁可少讲一个机制，也不要为了塞进一个结论而插入一段生硬的话。
6. **尾声回扣**：用目标人物自己的一句原话/比喻收尾，回扣开篇场景。

## 结构参考（非强制，可读性优先）

场景开篇（一个反常细节，埋下"为什么"的钩子）→ 童年/原点（不点破，只铺事实）
→ 把成年行为一条条接回原点，让读者自己看出那条线 → 用"时间点"检验那条线
（最有说服力的一击）→ 这条线的代价（盲区/那道结）→ 当下与去向 → 原话收尾。
全程不出现一个小标题里的术语，不出现一句"接下来分析……"式的过渡。

## 禁止事项

- 禁止任何方法论/心理学术语进入正文（见第 3 条清单）——这是本模版的第一红线
- 禁止在正文出现置信度、证据等级、证伪条件等元层语言——一律进文末附注
- 禁止虚构场景细节（天气、神态、未见于语料的对话）——只能复述语料中有记录的事实
- 禁止直接引用未通过引文校验的引语（查 `.verification.md`），意译需去引号
- 禁止把"低置信/推测"级结论在正文写成断言——正文用"大概""也许""讲得通"
  等口语化软化词承载不确定性，精确的置信度分层放文末附注

## 文末固定附注

正文之后用 `---` 分隔，附注标题用平实措辞（如"关于这篇文章"），**不要**用
"阅读说明""方法论附录"这类生硬标签。附注里可以、也应该出现正文严禁的元层语言
（成因假设、证伪判据、置信度分层、来源评分），这是它存在的意义——
把严谨从正文里拿出来单独安放，好让正文干净。固定包含四块：

```
### 关于这篇文章
- 核心假设的地位：本文核心是一个关于成因的假设而非定论；它为何可信
  （用同一条线索贯穿 N 个行为 + 命中时间点），一句话说清
- 它会怎样被证伪：写出 1 条具体的、可观察的反证条件
- 证据与置信度：哪些是事实记录（高）、哪些是推断（中/低）、最大资料缺口在哪
- 主要资料来源 + 技术版指引 + 引文校验提示（意译未逐字引用）
```

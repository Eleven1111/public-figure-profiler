# 公开人物心理侧写

对任意公开人物进行系统性认知与行为分析。基于多来源公开语料，输出可执行的侧写报告。

**核心原则：** 每个结论必须有原始文本引证；跨情境一致出现的才是稳定特征。

## 适用场景

- 「帮我分析 [人物名]」「给 [人物名] 做个侧写/画像」
- 「[人物名] 这个人的决策风格是什么」「他在压力下会怎么反应」
- 投资尽调、谈判准备、学术研究、媒体核查

## 执行流程

启动后依次询问（每次一个问题，等用户回答后再问下一个）：

**问题 1：** 分析目的是什么？
- 投资尽调（侧重决策风格和风险偏好）
- 谈判准备（侧重冲突处理和红线）
- 学术研究（全面分析）
- 媒体核查（侧重公开叙事一致性）
- 其他（请说明）

**问题 2：** 分析深度？
- **Quick Mode**：3个核心发现 + 行为预测，约15分钟
- **Deep Mode**：完整侧写报告 + JSON 结构化数据

**问题 3：** 你是否有额外语料？（访谈文字稿、演讲实录、文章等）
- 有 → 请粘贴文本或提供文件路径
- 没有 → 直接开始，我会通过网络搜索获取

收集到以上信息后加载分析框架：

```
Read references/codebook.md
Read references/output-schema.md
```

然后从 Step 0 开始执行分析流程（详见 codebook.md 和 agent/AGENT.md 中的步骤定义）。

## 语料获取策略（按优先级）

1. **用户提供** → 直接进入分析
2. **WebSearch + WebFetch** → 优先搜索播客全文稿、听证会记录、长篇深访
3. **YouTube 转录** → 对视频链接用 youtube-transcript-api 提取字幕
4. **用户手动补充** → 若音视频无可用转录，提示用户提供

对中文人物：额外搜索虎嗅、36氪、晚点 LatePost 的深访报道。

## 输出

- **Quick Mode**：纯文字，3个核心发现 + 行为预测
- **Deep Mode**：完整 Markdown 报告 + JSON（见 references/output-schema.md 中的模板）

所有结论使用「在公开表达中呈现出……倾向」等措辞，不做临床诊断。

## 用户可补充内容

分析过程中，用户可随时补充：
- 额外语料文本
- 时间节点信息（「他在2023年经历了X」）
- 特定问题（「我想重点了解他对监管的态度」）
